# Food Safety Guide

## 프로젝트 소개

Food Safety Guide는 사용자의 질환, 복용 약물, 알레르기 정보를 바탕으로 입력한 음식 또는 식단이 안전한지 안내해주는 LLM 기반 식단 안전 가이드 서비스입니다.

사용자는 자신의 건강 정보를 선택하고, 자연어로 식단을 입력할 수 있습니다.  
서비스는 입력된 정보를 바탕으로 식단을 `안전`, `주의`, `위험`, `정보 부족` 중 하나로 분류하고, 각 음식별 이유와 권장 행동을 제공합니다.

본 서비스는 의료 진단이나 처방을 목적으로 하지 않으며, 참고용 식단 안전 안내를 제공하는 프로토타입입니다.

---

## 주요 기능

- 질환 정보 선택
- 복용 약물 정보 선택
- 알레르기 정보 선택
- 음식 또는 식단 자연어 입력
- OpenAI API 기반 식단 위험도 분석
- 전체 식단 판정 제공
- 음식별 판정, 이유, 권장 행동 제공
- 의료 진단 및 처방 방지를 위한 안전 문구 제공

---

## 위험도 분류 기준

서비스는 식단 분석 결과를 다음 4단계로 분류합니다.

| 분류 | 의미 |
|---|---|
| 안전 | 현재 입력 정보 기준 특별한 위험 요소가 낮은 경우 |
| 주의 | 질환, 약물, 알레르기와 관련해 주의가 필요한 경우 |
| 위험 | 알레르기 또는 약물 상호작용 등으로 섭취를 피해야 할 가능성이 높은 경우 |
| 정보 부족 | 음식 성분이나 정보가 불확실하여 판단이 어려운 경우 |

프롬프트 설계 원칙

본 프로젝트는 의료 정보와 관련된 서비스를 다루기 때문에 다음 원칙을 프롬프트에 포함합니다.

의료 진단이나 처방을 하지 않는다.
약 복용 중단이나 용량 변경을 지시하지 않는다.
확실하지 않은 음식 성분은 추측하지 않고 정보 부족으로 분류한다.
알레르기와 직접 관련된 성분이 음식명에 명확히 포함되면 위험으로 분류한다.
결과는 반드시 JSON 형식으로 출력한다.
필요한 경우 의사 또는 약사와 상담하도록 안내한다.
예시 사용 흐름
사용자가 질환, 복용 약물, 알레르기 정보를 선택합니다.
사용자가 자연어로 식단을 입력합니다.
React 프론트엔드가 FastAPI 백엔드의 /analyze API로 요청을 보냅니다.
FastAPI는 입력값을 프롬프트에 포함하여 OpenAI API를 호출합니다.
OpenAI API는 식단 위험도 분석 결과를 JSON 형식으로 반환합니다.
React는 결과를 카드 형태로 사용자에게 보여줍니다.
주의사항

본 서비스는 의료 진단, 처방, 복약 지시를 제공하지 않습니다.
결과는 참고용 안내이며, 실제 건강 상태나 약물 복용과 관련된 판단은 반드시 의사 또는 약사와 상담해야 합니다.

개발 목적

본 프로젝트는 LLM을 활용하여 개인 건강 조건을 반영한 식단 안전 안내 서비스를 구현하는 것을 목표로 합니다.
기존의 칼로리 중심 식단 관리 서비스와 달리, 질환, 복용 약물, 알레르기 정보를 고려하여 사용자가 특정 음식을 섭취할 때 주의해야 할 점을 쉽게 이해할 수 있도록 돕는 데 초점을 둡니다.

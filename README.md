# Food Safety Guide

사용자의 질환, 복용 약물, 알레르기 정보를 바탕으로 입력한 음식 또는 식단의 섭취 위험도를 분석하는 AI 기반 식단 안전 가이드 프로젝트입니다.

사용자는 웹 화면에서 건강 정보를 선택하고 음식 정보를 자연어로 입력할 수 있으며, 백엔드는 OpenAI API를 활용해 식단 안전성 분석 결과를 JSON 형식으로 반환합니다. 프런트엔드는 이 결과를 전체 판정 카드와 음식별 분석 카드로 표시합니다.

> 본 서비스는 의료 진단이나 처방을 대체하지 않으며, 참고용 정보 제공을 목적으로 합니다.

---

## 프로젝트 개요

Food Safety Guide는 개인의 건강 조건을 고려한 식단 안전성 분석 서비스입니다.

일반적인 식단 정보 제공 서비스와 달리, 다음과 같은 사용자 정보를 함께 고려합니다.

- 질환 정보
- 복용 약물 정보
- 알레르기 정보
- 사용자가 입력한 음식 또는 식단

예를 들어 사용자가 `새우 알레르기`를 선택하고 `새우탕이랑 라면 먹어도 돼?`라고 입력하면, AI는 새우탕을 알레르기 위험으로 판단하고 라면은 나트륨 섭취 측면에서 주의가 필요하다고 분석할 수 있습니다.

---

## 주요 기능

- 질환 선택
- 복용 약물 선택
- 알레르기 선택
- 음식 또는 식단 자연어 입력
- AI 기반 식단 안전성 분석
- 전체 식단 위험도 판정
- 음식별 위험도, 판단 이유, 권장 행동 제공
- 의료 진단 및 처방이 아님을 알리는 면책 문구 제공
- 위험도별 결과 카드 표시

---

## 위험도 분류 기준

분석 결과는 다음 4단계로 분류됩니다.

| 위험도 | 설명 |
|---|---|
| 안전 | 입력된 건강 정보 기준으로 특별한 위험 요소가 낮은 경우 |
| 주의 | 질환, 약물, 알레르기와 관련해 섭취 시 주의가 필요한 경우 |
| 위험 | 명확한 알레르기 성분, 심각한 약물-음식 상호작용, 질환상 피해야 할 가능성이 높은 경우 |
| 정보 부족 | 음식 성분이나 건강 정보가 불충분해 안전성을 판단하기 어려운 경우 |

전체 위험도는 음식별 판정 중 가장 높은 위험도를 기준으로 결정됩니다.

위험도 우선순위는 다음과 같습니다.

```text
위험 > 주의 > 정보 부족 > 안전
```

## Frontend 설명

프런트엔드는 HTML 기반 웹 화면으로 구성되어 있습니다.

화면은 다음 영역으로 구성됩니다.

1. 서비스 제목 및 설명
2. AI 윤리 원칙 안내 배너
3. 질환 선택 영역
4. 복용 약물 선택 영역
5. 알레르기 선택 영역
6. 음식 입력창
7. 식단 안전성 분석 버튼
8. 분석 결과 카드
9. 오류 메시지 영역

### 선택 가능한 질환

```text
없음, 고혈압, 당뇨, 신장질환, 통풍
```

### 선택 가능한 복용 약물

```text
없음, 와파린, 스타틴, MAOI 항우울제, 갑상선약
```

### 선택 가능한 알레르기

```text
없음, 땅콩, 우유, 계란, 새우
```

사용자가 선택한 값은 JavaScript 상태 객체에 저장되며, 분석 버튼을 누르면 백엔드의 `/analyze` API로 요청이 전송됩니다.

```javascript
const API_URL = "http://localhost:8000/analyze";
```

## Backend 설명

백엔드는 FastAPI로 구현되어 있으며, 핵심 API는 `/analyze`입니다.

### Health Check

```http
GET /
```

응답 예시:

```json
{
  "message": "Diet guide backend is running"
}
```

### 식단 분석 API

```http
POST /analyze
```

요청 데이터는 다음 형식입니다.

```json
{
  "conditions": ["고혈압"],
  "medications": ["없음"],
  "allergies": ["새우"],
  "foodInput": "나 오늘 새우탕이랑 라면 먹어도 돼?"
}
```

백엔드는 입력된 사용자 정보를 프롬프트에 포함하여 OpenAI API에 전달하고, 분석 결과를 JSON 형식으로 반환합니다.

## 응답 JSON 구조

프런트엔드는 백엔드 응답이 다음 구조라고 가정하고 결과를 화면에 표시합니다.

```json
{
  "overallRiskLevel": "위험",
  "summary": "전체 식단에 대한 요약입니다.",
  "items": [
    {
      "foodName": "새우탕",
      "riskLevel": "위험",
      "reason": "새우 알레르기가 있는 사용자에게 새우가 포함된 음식은 알레르기 반응을 일으킬 수 있습니다.",
      "recommendation": "섭취하지 않는 것이 좋습니다. 정확한 판단을 위해 의사, 약사 또는 영양사와 상담하시기 바랍니다."
    },
    {
      "foodName": "라면",
      "riskLevel": "주의",
      "reason": "라면은 일반적으로 나트륨 함량이 높아 고혈압 관리에 불리할 수 있습니다.",
      "recommendation": "국물 섭취를 줄이고 자주 먹지 않는 것이 좋습니다. 개인 상태에 맞는 섭취 방법은 전문가와 상담하시기 바랍니다."
    }
  ],
  "disclaimer": "본 결과는 의료 진단이 아니며, 정확한 판단을 위해 의사, 약사 또는 영양사와 상담하시기 바랍니다."
}
```

---

## JSON 필드 설명

| 필드 | 설명 |
|---|---|
| overallRiskLevel | 전체 식단 위험도 |
| summary | 전체 분석 요약 |
| items | 음식별 분석 결과 배열 |
| disclaimer | 의료 면책 문구 |

`items` 배열의 각 항목은 다음 필드를 가집니다.

| 필드 | 설명 |
|---|---|
| foodName | 음식명 |
| riskLevel | 음식별 위험도 |
| reason | 판단 이유 |
| recommendation | 권장 행동 |

## 프롬프트 설계 원칙

프롬프트는 식단 안전 안내를 위해 다음 원칙을 따릅니다.

1. 의료 진단이나 처방을 하지 않습니다.
2. 약 복용 중단이나 용량 변경을 지시하지 않습니다.
3. 음식과 건강 정보의 관계가 불확실하면 추측하지 않고 `정보 부족`으로 분류합니다.
4. 알레르기와 직접 관련된 성분이 음식명에 명확히 포함되면 `위험`으로 분류합니다.
5. `주의`, `위험`, `정보 부족` 결과에는 전문가 상담 권고를 포함합니다.
6. 사용자에게 보이는 설명은 한국어로 작성합니다.
7. 응답은 반드시 정해진 JSON 구조를 따릅니다.

## 실행 방법

### 1. 저장소 클론

```bash
git clone https://github.com/kbs041216/Food_Safety.git
cd Food_Safety
```

---

### 2. 백엔드 실행

백엔드 폴더로 이동합니다.

```bash
cd backend
```

필요한 패키지를 설치합니다.

```bash
pip install fastapi uvicorn python-dotenv openai
```

`.env` 파일을 준비합니다.

```env
OPENAI_API_KEY=your_api_key
OPENAI_MODEL=your_model_name
```

FastAPI 서버를 실행합니다.

```bash
uvicorn main:app --reload --port 8000
```

정상 실행 여부는 브라우저에서 아래 주소로 확인할 수 있습니다.

```text
http://localhost:8000/
```

정상 실행 시 다음과 같은 응답이 표시됩니다.

```json
{
  "message": "Diet guide backend is running"
}
```

---

### 3. 프런트엔드 실행

새 터미널을 열고 프로젝트 루트 폴더로 이동합니다.

```bash
cd Food_Safety
```

HTML 파일을 브라우저에서 안정적으로 실행하기 위해 정적 서버를 실행합니다.

```bash
python -m http.server 5173
```

브라우저에서 다음 주소로 접속합니다.

```text
http://localhost:5173/food_safety_analyzer.html
```

---

## CORS 설정

백엔드는 프런트엔드 요청을 허용하기 위해 다음 주소를 허용합니다.

```python
allow_origins=["http://localhost:5173"]
```

따라서 프런트엔드는 `localhost:5173`에서 실행하는 것이 좋습니다.

## 주의사항

이 프로젝트는 의료 판단을 자동화하기 위한 서비스가 아니라, 사용자가 식단 선택 시 참고할 수 있는 안전 안내 도구입니다.

AI 응답은 실제 의학적 판단과 다를 수 있으며, 특히 알레르기, 만성질환, 약물 복용과 관련된 식단 결정은 반드시 의사, 약사 또는 영양사와 상담해야 합니다.
